## Proxy
> 알아두면 좋은 점: Next.js 16부터 미들웨어는 목적을 더 잘 반영하기 위해 이제 Proxy라고 불립니다. 기능은 동일하게 유지됩니다.

프록시를 사용하면 요청이 완료되기 전에 코드를 실행할 수 있습니다. 그런 다음 들어오는 요청에 따라 요청 또는 응답 헤더를 다시 작성, 리디렉션, 수정하거나 직접 응답하여 응답을 수정할 수 있습니다.

### Use cases
프록시가 효과적인 몇 가지 일반적인 시나리오는 다음과 같습니다:

- 모든 페이지 또는 페이지 하위 집합의 헤더 수정
- A/B 테스트 또는 실험을 기반으로 다른 페이지에 다시 작성
- 들어오는 요청 속성을 기반으로 하는 프로그래밍 방식 리디렉션

간단한 리디렉션의 경우 먼저 `next.config.ts`의 [리디렉션](https://nextjs.org/docs/app/api-reference/config/next-config-js/redirects) 구성을 사용하는 것을 고려하세요. 요청 데이터나 더 복잡한 로직에 액세스해야 하는 경우 프록시를 사용해야 합니다.

프록시는 느린 데이터 가져오기를 위한 것이 아닙니다. 프록시는 권한 기반 리디렉션과 같은 [낙관적 검사](https://nextjs.org/docs/app/guides/authentication#optimistic-checks-with-proxy-optional)에 도움이 될 수 있지만 전체 세션 관리 또는 권한 부여 솔루션으로 사용해서는 안 됩니다.

`options.cache`, `options.next.revalidate` 또는 `options.next.tags`와 함께 가져오기를 사용하면 프록시에 아무런 영향이 없습니다.

### Example
프록시 기능을 기본 내보내기 또는 명명된 `프록시` 내보내기로 내보낼 수 있습니다:

proxy.ts
```ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'
 
// This function can be marked `async` if using `await` inside
export function proxy(request: NextRequest) {
  return NextResponse.redirect(new URL('/home', request.url))
}
 
// Alternatively, you can use a default export:
// export default function proxy(request: NextRequest) { ... }
 
export const config = {
  matcher: '/about/:path*',
}
```

`matcher` 구성을 사용하면 특정 경로에서 실행되도록 프록시를 필터링할 수 있습니다. 경로 일치에 대한 자세한 내용은 [Matcher](https://nextjs.org/docs/app/api-reference/file-conventions/proxy#matcher) 설명서를 참조하세요.

[프록시 사용](https://nextjs.org/docs/app/guides/backend-for-frontend#proxy)에 대해 자세히 알아보거나 `프록시` [API 참조](https://nextjs.org/docs/app/api-reference/file-conventions/proxy)를 참조하세요.
