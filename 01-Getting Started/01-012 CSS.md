# CSS
<span style="color: #808080;">Last updated March 20, 2026</span>

Next.js는 CSS를 사용하여 애플리케이션의 스타일을 지정하는 여러 가지 방법을 제공합니다:

- [Tailwind CSS](https://nextjs.org/docs/app/getting-started/css#tailwind-css)
- [CSS Modules](https://nextjs.org/docs/app/getting-started/css#css-modules)
- [Global CSS](https://nextjs.org/docs/app/getting-started/css#global-css)
- [External Stylesheets](https://nextjs.org/docs/app/getting-started/css#external-stylesheets)
- [Sass](https://nextjs.org/docs/app/guides/sass)
- [CSS-in-JS](https://nextjs.org/docs/app/guides/css-in-js)

## Tailwind CSS
[Tailwind CSS](https://tailwindcss.com/)는 사용자 정의 디자인을 구축하기 위해 하위 수준 유틸리티 클래스를 제공하는 유틸리티 우선 CSS 프레임워크입니다.

Tailwind CSS 설치:

```terminal
npm install -D tailwindcss @tailwindcss/postcss
```

`postcss.config.mjs` 파일에 PostCSS 플러그인을 추가하세요:

postcss.config.mjs
```js
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

전역 CSS 파일로 Tailwind 가져오기:

app/globals.css
```css
@import 'tailwindcss';
```

루트 레이아웃에 CSS 파일 가져오기:

app/layout.tsx
```ts
import './globals.css'
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

이제 애플리케이션에서 Tailwind의 유틸리티 클래스를 사용할 수 있습니다:

app/page.tsx
```ts
export default function Page() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-between p-24">
      <h1 className="text-4xl font-bold">Welcome to Next.js!</h1>
    </main>
  )
}
```

> 알아두면 좋은 점: 매우 오래된 브라우저에 대해 더 광범위한 브라우저 지원이 필요한 경우 [Tailwind CSS v3 설정 지침](https://nextjs.org/docs/app/guides/tailwind-v3-css)을 참조하세요.

## CSS Modules
CSS 모듈은 고유한 클래스 이름을 생성하여 CSS 범위를 로컬로 지정합니다. 이를 통해 이름 충돌에 대한 걱정 없이 다른 파일에서 동일한 클래스를 사용할 수 있습니다.

CSS 모듈 사용을 시작하려면 확장자가 `.module.css`인 새 파일을 만들고 `app` 디렉터리 내의 모든 컴포넌트로 가져옵니다:

app/blog/blog.module.css
```css
.blog {
  padding: 24px;
}
```

app/blog/page.tsx
```ts
import styles from './blog.module.css'
 
export default function Page() {
  return <main className={styles.blog}></main>
}
```

## Global CSS
전역 CSS를 사용하여 애플리케이션 전체에 스타일을 적용할 수 있습니다.

`app/global.css` 파일을 생성하고 이를 루트 레이아웃으로 가져와 애플리케이션의 모든 경로에 스타일을 적용합니다:

app/global.css
```css
body {
  padding: 20px 20px 60px;
  max-width: 680px;
  margin: 0 auto;
}
```

app/layout.tsx
```ts
// These styles apply to every route in the application
import './global.css'
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

> 알아두면 좋은 점: 전역 스타일은 `app` 디렉토리 내의 모든 레이아웃, 페이지 또는 컴포넌트로 가져올 수 있습니다. 그러나 Next.js는 스타일시트에 대한 React의 내장 지원을 사용하여 Suspense와 통합하기 때문에 현재 충돌을 일으킬 수 있는 경로 사이를 탐색할 때 스타일시트를 제거하지 않습니다. Tailwind의 기본 스타일과 같은 진정한 전역 CSS에는 전역 스타일을 사용하고, 컴포넌트 스타일링에는 [Tailwind CSS](https://nextjs.org/docs/app/getting-started/css#tailwind-css)를 사용하고, 필요한 경우 사용자 정의 범위 CSS에는 [CSS 모듈](https://nextjs.org/docs/app/getting-started/css#css-modules)을 사용하는 것이 좋습니다.

## External stylesheets
외부 패키지에 의해 게시된 스타일시트는 함께 배치된 컴포넌트를 포함하여 `app` 디렉터리의 어느 곳에서나 가져올 수 있습니다:

app/layout.tsx
```ts
import 'bootstrap/dist/css/bootstrap.css'
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body className="container">{children}</body>
    </html>
  )
}
```

> 알아두면 좋은 점: React 19에서는 <link rel="stylesheet" href="..." />도 사용할 수 있습니다. 자세한 내용은 [React 링크 문서](https://react.dev/reference/react-dom/components/link)를 참조하세요.

## Ordering and Merging
Next.js는 스타일시트를 자동으로 청크(병합)하여 프로덕션 빌드 중에 CSS를 최적화합니다. CSS의 순서는 코드에서 스타일을 가져오는 순서에 따라 달라집니다.

예를 들어 `<BaseButton>`을 `page.module.css`보다 먼저 가져오므로 `base-button.module.css`는 `page.module.css`보다 먼저 주문됩니다:

page.tsx
```ts
import { BaseButton } from './base-button'
import styles from './page.module.css'
 
export default function Page() {
  return <BaseButton className={styles.primary} />
}
```

base-button.tsx
```ts
import styles from './base-button.module.css'
 
export function BaseButton() {
  return <button className={styles.primary} />
}
```

### Recommendations
CSS 순서를 예측 가능하게 유지하려면:

- 단일 JavaScript 또는 TypeScript 항목 파일에 CSS 가져오기를 포함해 보세요.
- 애플리케이션 루트에서 전역 스타일과 Tailwind 스타일시트를 가져옵니다.
- 유틸리티 클래스로 일반적인 디자인 패턴을 다루므로 대부분의 스타일링 요구 사항에 Tailwind CSS를 사용하세요.
- Tailwind 유틸리티가 충분하지 않은 경우 컴포넌트별 스타일에 CSS 모듈을 사용하세요.
- CSS 모듈에 일관된 명명 규칙을 사용하세요. 예를 들어 `<name>.tsx` 대신 `<name>.module.css`를 사용합니다.
- 중복 가져오기를 방지하려면 공유 스타일을 공유 컴포넌트로 추출하세요.
- ESLint의 [정렬 가져오기](https://eslint.org/docs/latest/rules/sort-imports)와 같이 가져오기를 자동 정렬하는 린터나 포맷터를 끄세요.
- `next.config.js`의 [cssChunking](https://nextjs.org/docs/app/api-reference/config/next-config-js/cssChunking) 옵션을 사용하여 CSS가 청크되는 방식을 제어할 수 있습니다.

## Development vs Production
- 개발(`next dev`)에서는 [빠른 새로 고침](https://nextjs.org/docs/architecture/fast-refresh)을 통해 CSS 업데이트가 즉시 적용됩니다.
- 프로덕션(`next build`)에서는 모든 CSS 파일이 자동으로 여러 축소 및 코드 분할 `.css` 파일로 연결되어 경로에 대해 최소한의 CSS가 로드되도록 합니다.
- 프로덕션에서는 JavaScript가 비활성화된 상태로 CSS가 계속 로드되지만 빠른 새로 고침을 위해서는 개발 시 JavaScript가 필요합니다.
- CSS 순서는 개발 중에 다르게 동작할 수 있으므로 항상 빌드(`next build`)를 확인하여 최종 CSS 순서를 확인하세요.
